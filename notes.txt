 A typical workflow for a simultaneous change:

  1. Edit files in blink/, commit and push there:
  cd blink
  # edit files
  git add -A && git commit -m "fix something"
  git push
  cd ..
  2. Then in portator, the submodule pointer has moved, so commit that:
  git add blink
  git commit -m "update blink submodule"

  The submodule pin in portator is just a commit hash — whenever you make commits inside blink/, you update the pin in portator by
  staging the blink directory.
